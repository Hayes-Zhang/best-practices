## Contributing examples

PRs welcome. Good examples should:

1. **Be real outputs**, not edited mockups — even with imperfections.
2. **Cover diverse topics**: backend, frontend, AI/ML, devops, design, growth.
3. **Show both success and failure modes** — low-confidence responses are as
   valuable as high-confidence ones because they demonstrate the plugin's
   honesty.
4. **Strip personal/project identifiers** before submitting.
